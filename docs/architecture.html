Filter Data Flow

FilterBar.vue (Time Period, Warehouse, Category, Status) → two-way bound to useFilters.js refs → view calls getCurrentFilters() → api.js builds query params (skips "all") → FastAPI route → apply_filters() / filter_by_month() → Pydantic-validated JSON → view ref → computed re-derives display data.

Because filter state is module-level (not per-component), every view shares the same selection with no prop drilling.

Known Gaps

Found while exploring the codebase — worth knowing before building on top of it:
  • api.js defines /api/tasks and /api/purchase-orders calls that do not exist in main.py yet — calling them 404s.
  • views/Backlog.vue is fully built but not wired into the router or nav in main.js / App.vue.
  • /api/demand and /api/backlog ignore the global filter bar server-side.
  • /api/inventory has no time-dimension filter (no month param).
